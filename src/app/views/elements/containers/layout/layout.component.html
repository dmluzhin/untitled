Контейнер

Класс .container может использоваться в любом контексте, но в основном зависит от родительского блока. Основные блоки, в которм используется .container:

  • .navbar
  • .hero
  • .section
  • .footer

Ширина контейнера для каждого брейкпоинта равна $device - (2 * $gap). Переменная $gap имеет стандартное значение в 32px, но может быть изменено.

Вот как будет вести себя контейнер при разных разрешениях:

  • При разрешении $desktop размер контейнера составит максимум 960px
  • При разрешении $widescreen размер контейнера составит максимум 1152px
  • При разрешении $fillhd размер контейнера составит макмимум 1344px
Здесь есть контейнер центрированный на разрешении $desktop. 
            {{container}}
Резиновый контейнер

Если не хотите, чтобы контейнер ограничивался максимальной шириной, зависимой от разрешения, но хотите оставить паддинги в 32px слева и справа, используйте класс .is-fluid:

Это резиновый контейнер с сохранением паддингов в 32 пикселя 
{{fluidContainer}}
Контейнеры с брейкпоинтами

С двумя модификаторами .is-widescreen и .is-fullhd, можно отображать полноразмерный контейнер до этих конкретных брейкпоинтов.

Этот контейнер занимает всю ширину родительского блока до брейкпоинта $widescreen. 
{{widescreenContainer}}
Этот контейнер занимает всю ширину родительского блока до брейкпоинта $fullscreen. 
{{fluidContainer}}
